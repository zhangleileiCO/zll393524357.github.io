关于git工作流

最近项目中使用复杂的git工作流模式，在这里加以整理

git工作流主要分为：集中式工作流，功能分支工作流，giflow工作流，forKing工作流

我们的项目目前主要是用gitflow和forking工作流的杂交版本

现整理如下（如有问题以后再改）

1.使用bitbucket进入对应公共repo fork一个项目

2.在 your work中找到刚刚fork的repo，然后将这个repo clone到开发电脑上（自动创建别名origin）

命令：git clone https://zhangleilei_cloud@bitbucket.org/zhangleilei_cloud/neptune.git

3.添加公共远程repo别名（upstream ）

命令：git remote add upstream https://zhangleilei_cloud@bitbucket.org/meta-sota/neptune.git

4.新建本地develop分支

命令：git checkout -b develop origin/develop

5.用公共仓库develop更新本地develop

命令：git pull upstream develop

6.新建开发分支

git checkout -b feature-xxxx develop

7.开发

git status
git add
git commit

8.开发完成之后pull upstream的develop分支，merge滞后代码

命令：git pull upstream develop 或者加上 git pull –rebase upstream develop

9.push代码到自己fork的仓库（origin）

git push origin feature-branch

10.在bitbucket上提交pull request

​ 合并fork的repo中的feature分支到公共repo的develop分支

11.删除feature

12.如果下次有新的开发任务，checkout到开发电脑上的develop分支，然后第5部开始。

流程还未经验证，在使用的时候如果有问题就上来修改

参考链接（https://github.com/xirong/my-git/blob/master/git-workflow-tutorial.md ）
